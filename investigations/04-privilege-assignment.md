# 🔐 SOC Investigation Report: Privilege Assignment Monitoring (Event ID 4672)

This report documents a simulated SOC investigation analyzing privileged logon events using Elastic Stack in a controlled lab environment.

---

## 📌 Summary

A privileged logon event (Event ID 4672) was detected, indicating that special administrative privileges were assigned to a user session. These events are important for monitoring potential privilege escalation but require contextual analysis to determine if they are legitimate or suspicious.

---

## 🖥️ Environment

- **SIEM:** Elastic Stack (Elasticsearch, Kibana, Fleet, Elastic Defend)  
- **Endpoint:** Windows 10 Virtual Machine  
- **User:** SYSTEM  
- **Host:** desktop-qucd21v  

---

## 🚨 Detection Details

### 🔹 Rule Name:
**Suspicious Privilege Assignment Detection**

### 🔹 Detection Logic:

```
event.code: 4672 AND NOT user.name: ("SYSTEM" OR "system")
```

### 🔹 MITRE ATT&CK Mapping:

- **Tactic:** Privilege Escalation (TA0004)  
- **Technique:** T1548 – Abuse Elevation Control Mechanism  

---

## 🔍 Investigation Steps

### 1. Log Analysis

Filtered privileged events using:

```
event.code: 4672
```

Observed privilege assignment events generated by the system.

---

### 2. Key Fields Observed

- **event.code:** 4672  
- **user.name:** SYSTEM  
- **host.name:** desktop-qucd21v  
- **event.category:** authentication  

---

### 3. Behavior Identification

- Privileged logon event generated  
- Special privileges assigned to the SYSTEM account  

---

## ⏱️ Timeline Reconstruction

```
1. System process initiated privileged operation  
2. Windows assigned special privileges to SYSTEM account  
3. Event ID 4672 generated  
4. Detection rule evaluated event  
```

---

## 🧠 Behavioral Analysis

### 🔹 Observed Behavior

- Privilege assignment to SYSTEM account  

### 🔹 Interpretation

- SYSTEM account inherently has high privileges  
- Event ID 4672 is frequently generated during normal system operations  
- This behavior is expected and not inherently malicious  

---

## ⚖️ Analysis & Findings

### 🔹 Indicators

- Privileged logon detected  
- No abnormal user activity observed  
- No suspicious preceding events  

### 🔹 Context Evaluation

- Event generated by SYSTEM account  
- No evidence of unauthorized privilege escalation  
- Behavior aligns with normal OS functionality  

---

## 🧭 MITRE ATT&CK Mapping

- T1548 – Abuse Elevation Control Mechanism  

---

## 🧾 Final Classification

```
Benign Activity → Normal System Behavior
```

---

## 🚀 SOC Analyst Decision

- **Severity:** Low  
- **Action Taken:** No escalation required  

### 🔹 Recommended Actions (Real Environment)

- Monitor for Event ID 4672 from non-SYSTEM users  
- Correlate with login activity (Event ID 4624 / 4625)  
- Investigate if preceded by unusual or suspicious behavior  

---

## 🧠 Key Learnings

- Event ID **4672** indicates privilege assignment during logon  
- It does not directly detect privilege escalation via UAC  
- SYSTEM account frequently generates benign events  
- Detection rules should exclude SYSTEM to reduce noise  
- Context and correlation are essential for accurate classification
